13. エラー処理(try/except)

CONCEPT

プログラムがエラーで止まらないようにする仕組みです。「もしここで失敗したら、こう対応する」という保険のようなものです。車のシートベルトのような役割です。

▶ コードを1行ずつ実行してみよう
example.pyPython
1process_payment("A001", 1500)
2 try: amount = 1500 > 0 → OK
3 print("注文A001: ¥1500を決済しました")
4 finally: print("決済処理を終了します")
5process_payment("A002", -100)
→ « 次の行 » ボタンを押して実行を開始
VARIABLES
実行すると変数の状態が表示されます
📚 読み方ガイド
コード読み方
try:「試しにやってみる」ブロックの開始
except ValueError as e:ValueError が出たらここに来る
except Exception as e:その他のエラーも全部拾う
finally:成功・失敗に関わらず必ず最後に実行
KEY POINT

try は「試しに実行する」範囲の開始。except は「もしエラーが起きたら」の対処法。finally は成功しても失敗しても必ず最後に実行される。エラーの種類ごとに処理を分けられる。
